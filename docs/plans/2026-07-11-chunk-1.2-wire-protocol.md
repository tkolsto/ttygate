## Critical pre-execution review

- **Roadmap/rewrite plan:** Produces only the contract and pure codecs consumed by Chunks 1.4–1.6. Ticket redemption is documented but not implemented.
- **Threat model:** Bounds are checked before parsing/copying; strict schemas prevent extension smuggling; hostile payloads are not reflected; terminal bytes remain opaque.
- **Directionality:** Separate APIs prevent client acceptance of `exit-status`/`error`/server `close` and server acceptance of `resize`/client `close`, even where a type name overlaps.
- **Lifecycle sufficiency:** Resize, client close request, server exit outcome, safe error, and terminal close notification cover later session/bridge/frontend needs without re-attach or speculative negotiation.
- **Drift prevention:** Both implementations consume identical valid/invalid fixture bytes and assert protocol constants. Any wire change requires specification and corpus changes that fail both suites.
- **Versioning:** Exact version matching is intentionally simple for v0.1; a future incompatible version adds parallel parsing rather than silently accepting unknown semantics.
