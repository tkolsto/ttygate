# Chunk 4.1 Docker and systemd Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Deliver reproducible, non-root Docker packaging and a hardened,
watchdog-enabled systemd service with executable cleanup-focused smoke tests.

**Architecture:** Build frontend assets and the Rust daemon in digest-pinned
builder stages, then copy only runtime artifacts into a fixed non-root Debian
runtime. Add a bounded daemon health-check command and native systemd readiness
and watchdog notification; validate packaging statically everywhere and
dynamically in Linux/Docker smoke jobs.

**Tech Stack:** Rust/Tokio, `sd-notify`, Docker/BuildKit, POSIX shell, systemd,
GitHub Actions.
