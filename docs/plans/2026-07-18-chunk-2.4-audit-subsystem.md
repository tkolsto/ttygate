# Chunk 2.4 Audit Subsystem Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement the complete structured append-only lifecycle audit subsystem tracked by #10 without logging terminal contents or credentials.

**Architecture:** A process-owned `AuditLog` opens the literal configured destination before application construction or binding, accepts only closed versioned event enums, and serializes each JSONL record under one mutex. HTTP, authentication, ticket, WebSocket, and supervised session paths emit stable attribution/denial records; any writer failure permanently denies new authority.

**Tech Stack:** Rust 2024, serde/serde_json, `time` RFC 3339 formatting, Unix append/no-follow file handling, axum listener `ConnectInfo`, Tokio supervised sessions, existing direct-TLS/trusted-proxy fixtures, Node frontend tests, and Playwright Chromium E2E.

**Issue reference:** Closes #10. Every implementation and documentation commit references #10.
