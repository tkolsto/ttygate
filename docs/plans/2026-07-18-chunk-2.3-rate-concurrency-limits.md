# Chunk 2.3 Rate and Concurrency Limits Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Bound session-request and authentication-failure rates and reserve
global/per-identity capacity at ticket issue time without weakening any
existing browser, transport, target, WebSocket, or PTY authority boundary.

**Architecture:** Add constant-space first-request-anchored fixed-window
limiters to `AppState`, keyed by authenticated identity or listener-supplied
peer IP. Extend the session manager's existing capacity authority with
move-only pending leases that tickets own and transfer atomically to live
session supervisors.

**Tech Stack:** Rust 2024, axum 0.8, Tokio 1, `std::sync::Mutex`,
`std::time::Instant`, existing cookie/ticket/session/WebSocket/PTY code,
TypeScript frontend tests, real direct-TLS and trusted-proxy fixtures.

**Issue:** Use `Refs #24` on every development commit. Use `Closes #24` only in
the final PR after every issue criterion is verified.

**Design:** `docs/plans/2026-07-18-chunk-2.3-rate-concurrency-limits-design.md`

## Task 2: Implement the deterministic bounded fixed-window limiter

**Files:**
- Create: `crates/ttygated/src/rate_limit.rs`
- Modify: `crates/ttygated/src/lib.rs`

**Intended private/public API:**

```rust
pub trait Clock: Send + Sync {
    fn now(&self) -> Instant;
}

pub struct FixedWindowLimiter<K> { /* clock, window, limit, capacity, buckets */ }

pub struct Attempt<K> { /* limiter, key, generation, committed */ }

pub enum LimitError {
    Exhausted { retry_after: Duration },
}

impl<K: Clone + Eq + Hash> FixedWindowLimiter<K> {
    pub fn new(limit: u32, window: Duration, capacity: usize) -> Self;
    pub fn with_clock(
        limit: u32,
        window: Duration,
        capacity: usize,
        clock: Arc<dyn Clock>,
    ) -> Self;
    pub fn begin(&self, key: K) -> Result<Attempt<K>, LimitError>;
}

impl<K: Clone + Eq + Hash> Attempt<K> {
    pub fn commit(mut self);
    pub fn rollback(mut self);
}
```

Dropping an uncommitted attempt rolls it back. Session requests call
`commit`; authentication failures retain only on failure.
