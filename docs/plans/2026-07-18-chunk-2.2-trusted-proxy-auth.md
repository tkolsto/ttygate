# Chunk 2.2 Trusted Reverse-Proxy Authentication Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Authenticate only configured identity headers supplied by real
trusted socket peers, transition that identity into ttygate's secure browser
cookie, and preserve identity through tickets, WebSockets, sessions, and PTY
ownership.

**Architecture:** Add a trusted-proxy implementation of the existing
`AuthProvider` interface. It validates listener-supplied peer addresses on
every authority request, reads exactly one configured header only while
establishing an opaque server-side cookie session, and otherwise reuses the
existing identity-bound ticket/session pipeline. Trusted-proxy transport serves
plaintext only inside the protected proxy-to-daemon boundary; a real
TLS-terminating test proxy proves HTTPS/WSS behavior without trusting forwarded
address headers.

**Tech Stack:** Rust 2024, axum 0.8, Tokio 1, `http::HeaderMap`, `ipnet::IpNet`,
rustls test certificates, raw HTTP integration fixtures, tokio-tungstenite,
existing PTY/session manager, TypeScript frontend checks.

**Issue:** Use `Refs #9`; do not close #9 because persistent audit attribution
belongs to future Chunk 2.4.

**Design:** `docs/plans/2026-07-18-chunk-2.2-trusted-proxy-auth-design.md`
