# Chunk 2.1 Production Gating and Direct TLS Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make unsafe production and development exposure fail before application construction or listener binding, and serve the unchanged M1 frontend/API/WebSocket authority path over one direct-rustls listener.

**Architecture:** Extend the closed TOML schema with provider-specific authentication and mutually exclusive plaintext, direct-TLS, and future trusted-proxy transport types. Separate combination validation, TLS material preparation, application construction, binding, and serving so tests can prove fail-before-serve ordering. Use axum-server/rustls for the listener and test it with an ephemeral test CA through normal HTTPS and WSS verification.

**Tech Stack:** Rust 2024, serde/TOML, axum 0.8, Tokio 1, axum-server 0.8, rustls 0.23, ipnet 2, rcgen test certificates, tokio-rustls test clients, existing xterm.js frontend and real PTY fixtures.

**Issue:** Use `Refs #8` unless every acceptance criterion is genuinely complete.

**Design:** `docs/plans/2026-07-18-chunk-2.1-production-gating-design.md`
